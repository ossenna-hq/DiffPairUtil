# Documentation

## Purpose

- Capture requirements before implementation and keep durable project documentation current.

## Ownership

- Owns files under `docs/`, including requirements and architecture documentation.

## Local Contracts

- Requirements describe functional intent.
- Architecture documentation describes implementation boundaries and interfaces.
- Every documentation folder must include an `Index.md` that explains contained files and child folders.

## Work Guidance

- Keep calculation assumptions, supplier data provenance, and deployment workflow notes documented when they change.

## Verification

- Review affected `Index.md` files after documentation changes.

## Child DOX Index

- `requirements/Index.md` - Functional outcomes and acceptance expectations.
- `architecture/Index.md` - Application, calculation, data, and deployment architecture.
